## 三、论文写作核心规则

### 3.1 写作顺序（铁律）

```
第2章（理论基础）→ 第4章（基准分析）→ 第3章（建模网格）
    → 第5章（参数优化）→ 第1章（绪论）→ 第6章（结论）
```

**原因**：先写不依赖仿真数据的理论章，再写有数据的分析章，绪论最后写（知道全貌后最好定位）。

### 3.2 语言风格

- 学术严谨，避免口语化、主观判断
- 专业术语首次出现附英文全称
- 不得编造文献、公式、数据
- 他人文献结果不得写成自己的仿真结果
- 拒绝 AI 凭空生成的参考文献（必须 DOI/期刊/作者可查证）

### 3.3 引用规则

- 正文用顺序编码制上角标：[1]、[2,3]、[1-3]
- 参考文献按正文首次出现顺序编号 1~N（不保留数据库稀疏编号）
- 最终参考文献按 GB/T 7714-2015 格式
- 引用内容必须可追溯（来自 PDF 原文、已提取 Markdown 或已核查文献实体页）
- 不得直接复制原文，必须理解后改写

### 3.4 学术合规

- 仿真结果归仿真结果，文献结论归文献结论，分清楚
- "结果与分析"必须是基于参考文献的对比归纳分析
- 公式必须可追溯来源（EQ_SOURCES 字典）
- 所有关键数据可复现（提供脚本 + 输入文件）

## 四、python-docx 论文生成 — 完整规范

### 4.1 页面设置（以江苏大学 2026 版为例）

```
纸张：A4（21.0cm × 29.7cm）
页边距：上 2.5 / 下 2.0 / 左 2.5 / 右 2.0 cm（非对称）
左侧装订
```

**注意**：必须以学校 PDF 规范为准，模板可能有滞后。

### 4.2 样式系统

| 样式 | 字体 | 字号 | 加粗 | 对齐 | 行距 |
|------|------|------|------|------|------|
| 一级标题 | 黑体 | 四号 14pt | 是 | 居中 | 1.5 |
| 二级标题 | 宋体 | 小四 12pt | 是 | 居左 | 1.5 |
| 三级标题 | 宋体 | 小四 12pt | 是 | 居左 | 1.5 |
| 正文 | 宋体(中文) + Times New Roman(西文) | 小四 12pt | 否 | 两端对齐 | 1.5 |

**标题颜色必须显式设为黑色 `RGBColor(0,0,0)`**——Word 默认 Heading 是蓝色！

### 4.3 公式（三大铁律）

1. **用 OMML 公式编辑器 + 制表位编号**，严禁用表格包裹公式
2. **公式文本绝对无空格**：OMML `m:t` 中不含任何 ASCII 空格
3. **必须用正确的 OMML 元素结构**：
   - 分式：`<m:f>`，禁止用 "/"
   - 下标：`<m:sSub>`，禁止用 "_"
   - 上标：`<m:sSup>`，禁止用 "^"
   - 根号：`<m:rad>`，禁止用 Unicode √

**公式制表位计算**：
```
文本区宽度 = 21 - 2.5 - 2.0 = 16.5 cm
居中制表位 = 8.25 cm（CENTER）
右对齐制表位 = 16.5 cm（RIGHT）
```

**OMML 字体**：必须显式设 `Times New Roman`（OMML 默认 Cambria Math）：
```python
for om in para._element.findall('.//' + qn('m:oMath')):
    for r in om.findall('.//' + qn('m:r')):
        rpr = r.find(qn('m:rPr'))
        if rpr is None: rpr = etree.SubElement(r, qn('m:rPr'))
        font = etree.SubElement(rpr, qn('m:font'))
        font.set(qn('m:ascii'), 'Times New Roman')
        font.set(qn('m:hAnsi'), 'Times New Roman')
        font.set(qn('m:cs'), 'Times New Roman')   # ← 遗漏会导致括号回退 Cambria
```

### 4.4 图/表标题

- 图题：宋体五号 10.5pt，居中，置于图下，"图 2-1 …"
- 表题：宋体五号 10.5pt，加粗，居中，置于表上，"表 2-1 …"
- 表格用三线表格式（高校论文标准）
- 所有表格单元格段落首行缩进清零：`first_line_indent=Pt(0)`

### 4.5 上角标引用实现

```python
def body(doc, text):
    para = doc.add_paragraph(style='Normal')
    # 正则拆分 [1]、[2,3]、[1-3] → superscript run
    pattern = r'\[([^\]]+)\]'
    last = 0
    for m in re.finditer(pattern, text):
        if m.start() > last:
            para.add_run(text[last:m.start()])
        r = para.add_run('[' + m.group(1) + ']')
        r.font.superscript = True
        last = m.end()
    if last < len(text):
        para.add_run(text[last:])
```

## 五、核心陷阱清单（52 条经验浓缩）

### 生成阶段陷阱

1. 合并章节必须用 `doc.element.body` 级别复制，不能只复制 `doc.paragraphs`（会丢表格和图片）
2. `.doc` 模板先用 Word COM 转 `.docx`（python-docx 不读旧版 .doc）
3. 终稿必须做图表数量验收（tables 数 + drawings 数）
4. 公式不能在表格里
5. WSL 路径用 `/mnt/d/...`，不用 `D:\...`
6. `execute_code` 沙箱拒绝中文 docstring → 必须 `write_file` + `terminal`
7. 实际论文数据结构（4 章 vs 6 章）因课题类型而异

### 修改/修复陷阱

8. **反复修改同一 docx 会导致图片丢失**：每次从源文件 `shutil.copy2` 重新开始，所有修改在单次脚本中完成
9. post-hoc 修复必须先 `shutil.copy` → 在新文件上改
10. post-hoc 修复绝不能清空含图片段落的 run 列表（`_has_drawing` 保护）
11. 全文字体修复必须同时遍历 `paragraphs` + `tables`
12. OMML 段落不能整体跳过修复（只跳 OMML 内部的 run）
13. 图片插入必须在数据替换之后（否则替换逻辑破坏 `w:drawing`）
14. 删除批注需清理 6 处（comments.xml + document.xml + rels 等）

### 格式细节陷阱

15. 标题编号空格：一级标题空 2 格（"1  绪论"），二三级空 1 格（"1.1 研究"）
16. 页边距导致文本区宽度变化时，公式制表位必须同步调整
17. 封面已存在时不要重复插入（先检查前 15 段）
18. 用户确认的实验数据优先于论文已有数据（全面替换：正文+图题+表题+表格+摘要+结论）
19. matploblib 图片中 Unicode 上/下角标渲染为 □ → 用 LaTeX 数学表达式 `$...$`
20. docx 中图片可通过 ZIP 操作原地替换（不需 python-docx 的 rel 操作）
21. OMML 公式中禁止混入中文解释文字
22. 修改已有 docx 图表标题字体时，必须区分"标题行"和"引用行"
23. 特殊 Unicode 字符（³²°×·→）需强制设 Cambria Math 字体

### 论文内容陷阱

24. 论文事实修正禁止全文盲替 → 三阶段流程：关键词扫描 → 逐段逐 run 精准替换 → 二次扫描验证
25. 绪论中引用他人文献的描述不是本人实验，不应修改
26. 全文数据替换后必须双向验证（旧值残留=0 + 新值出现次数正确）
27. Excel 实验数据列索引必须先打印表头验证（合并单元格/多级表头）

- 来源：arXiv、Semantic Scholar、Web of Science

### 引用核查工作流

```

                                              ↓
正文引用 [n] ← 引用位置表 ← 按首次出现重排 1~N ← 参考文献表 GB/T 7714
```

### 参考文献格式（GB/T 7714-2015 顺序编码制）

- [J] 期刊论文：[序号] 作者. 题名[J]. 刊名, 年, 卷(期): 起止页码.
- [D] 学位论文：[序号] 作者. 题名[D]. 学校, 年.
- [M] 专著：[序号] 作者. 书名[M]. 出版地: 出版社, 年.

## 八、质量审计工作流

### 审计项目

1. **图片检查**：编号连续性、标题规范性、各章配图完整性
2. **公式检查**：编号完整性、OMML 字体、引用-定义匹配
3. **参考文献检查**：格式规范、引用-文献匹配、真实性可查证
4. **文本检查**：残缺口语化表达、连续逗号、缺失数据
5. **数据验证**：外特性数据一致性、百分数计算正确性

### 常见问题清单

- OLE 公式无编号标签（最常见）
- 建模章缺配图
- "附近，，"连续逗号（公式编辑器残留）
- WMF ≠ 实际插图（需用 `w:drawing` 统计）
- 引用未上角标（用 post-hoc 修复脚本）

## 十、CFD 仿真自动化要点

### PyFluent 脚本模板

```python
import ansys.fluent.core as pyfluent
s = pyfluent.launch_fluent(
    precision='double', processor_count=8,
    mode='solver', meshing_mode=False
)
s.file.read_mesh(file_name='mesh.msh.h5')
s.tui.define.models.viscous.kw_sst()       # 湍流模型
s.tui.define.boundary_conditions.fluid('zhuandon', 'yes', 'frame_motion', ...)
s.solution.initialization.hybrid_initialize()
s.solution.run_calculation.iterate(iter_count=1500)
s.file.write_case_data(file_name='result.cas.h5')
```

### 后处理导出

- 云图：`s.results.graphics.contour` → matplotlib 重绘
- CSV 导出：中截面数据用 `s.tui.report.surface_integrals`
- 关键指标：H（扬程）、η（效率）、Tz（扭矩）、ΔP（进出口总压差）

### 命名规范

```
网格无关性：coarse_400k, medium_130w, fine_2500k
Q 扫描：Q{8,12,16,20,24} m³/h
参数优化：CASE-{01..15}
图片：{网格}-{工况}-{类型}.png
```
